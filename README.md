# Учебный проект по дисциплине Прикладной Python. Сервис по сокращению ссылок

Выполнена разработка сервиса, где пользователь отправляет свою длинную ссылку на любой ресурс, а сервис создаёт для неё уже короткую (задётся пользователем или генерируется сервисом), переход по которой перенаправялет на исходную пользовательсую ссылку

Подразумевается, что юзер регистрируется и логинится в сервисе, получает свой уникальный токен, создаёт ссылку/ссылки, которые может переименовывать, удалять и смотреть статистику 

В проекте реализованая регистрация пользователей, логированрие переходов по ссылкам, поиск укороченных ссылок по их длинной версии, возможность задавать срок жизни ссылок и их ручное удаление

> У пользователя есть имя, email, хеш пароля и дата создания

> У ссылки есть короткий код, исходный URL, пользовательское/сгенерированное сервисом название, даты создания и обновления, срок жизни, счётчик переходов, дата последнего перехода, флаг активности, причина деактивации и владелец


Почти вся логика лежит в одном файле `app/main.py`

В качестве базы данных используется SQLite, а для кэша используется Redis (подниматеся через docker compose, без него работа осуществляется без кеширования)

### Локальный запуск
Устновка окружения:

```.\.venv\Scripts\Activate.ps1```

Установка зависимостей:

```pip install -r requirements.txt```

Запуск приложения:

```uvicorn app.main:app --reload```

Документация доступна по дефолт адресу:
```http://127.0.0.1:8000/docs```

Запуск проекта:

```docker compose up --build```

Отдельный запуск Redis:

```docker compose up -d redis```


Основные ручки:

| Метод  | Ручка                         | Назначение                                          |
| ------ | -------------------------------- | --------------------------------------------------- |
| POST   | `/auth/register`                 | Регистрация юзера                           |
| POST   | `/auth/login`                    | Логин юзера |
| GET    | `/auth/me`                       | Инфа о текущем юзере        |
| POST   | `/links/shorten`                 | Создание короткой ссылки                             |
| GET    | `/links/{short_code}`            | Редирект на исходную ссылку                     |
| PUT    | `/links/{short_code}`            | Обновление исходной ссылки                              |
| DELETE | `/links/{short_code}`            | Удаление ссылки                                      |
| GET    | `/links/{short_code}/stats`      | Статистика по ссылки                     |
| GET    | `/links/search?original_url=...` | Поиск ссылки                        |
| GET    | `/links/expired-history`         | История истёкших ссылок                  |
| POST   | `/maintenance/cleanup`           | Запуск очистики истёкших/неиспользованных ссылок    |



Для проверки проекта есть quick_test в папке `tests`:

```pytest -q tests -p no:cacheprovider```

Реализована возможность вручную запускать очистку ссылок из командой строки

```python -m app.tasks.run_cleanup```

## Нагрузочное тестирование

Для нагрузочного тестирования используется *Locust*

Команда запуска -> ```locust -f locustfile.py --host http://127.0.0.1:8000```

Сценарий находится в `locustfile.py` и имитирует пользователя, который регистрируется, логинится, создаёт короткие ссылки, получает статистику и переходит по короткой ссылке
